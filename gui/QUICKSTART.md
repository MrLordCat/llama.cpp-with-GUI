# 🚀 Быстрый старт - GUI для llama.cpp

## За 5 минут до первого запуска!

### Шаг 1: Установка зависимостей (1 минута)

```bash
# Windows PowerShell
pip install -r gui\requirements-gui.txt
```

### Шаг 2: Запуск GUI (10 секунд)

```bash
python gui\llama_gui.py
```

### Шаг 3: Первоначальная настройка (2 минуты)

1. **Проверьте систему**:
   - Откройте вкладку "💻 Информация о системе"
   - Посмотрите, какой GPU обнаружен
   - Запомните рекомендуемый backend

2. **Соберите llama.cpp** (если ещё не собрано):
   - Откройте вкладку "🔧 Сборка и настройка"
   - Выберите backend для вашего GPU:
     - **NVIDIA GPU** → CUDA
     - **AMD GPU** → Vulkan
     - **Нет GPU** → CPU
   - Нажмите "⚙️ Конфигурировать"
   - Подождите 10-20 секунд
   - Нажмите "🔨 Собрать"
   - Подождите 5-15 минут (в зависимости от системы)

### Шаг 4: Загрузка модели (2-10 минут, зависит от размера)

1. Откройте вкладку "📥 Загрузка моделей"
2. Нажмите "⭐ Популярные" (загрузятся топ-модели)
3. Дважды кликните на модель (рекомендую начать с небольшой):
   - **Для начала**: Mistral-7B или Llama-2-7B
   - **Квантизация**: Q4_K_M (хороший баланс)
4. Выберите файл с Q4_K_M в названии
5. Нажмите "📥 Скачать модель"
6. Дождитесь окончания загрузки

### Шаг 5: Запуск! (30 секунд)

1. Откройте вкладку "🚀 Запуск сервера"
2. Выберите загруженную модель из списка
3. Оставьте режим "🌐 Веб-интерфейс"
4. Нажмите "▶️ Запустить сервер"
5. Подождите ~10-30 секунд пока модель загрузится
6. Автоматически откроется браузер с чатом!

## 🎉 Готово!

Теперь вы можете:
- Общаться с моделью в веб-интерфейсе
- Переключиться в API режим для интеграции с VSCode
- Загружать другие модели
- Экспериментировать с параметрами

## 💡 Полезные советы

### Если GPU слабый или мало VRAM:
- Уменьшите "GPU слоёв" до 10-20
- Выберите меньшую модель (7B вместо 13B)
- Используйте квантизацию Q4 или Q2

### Если нет GPU:
- Выберите backend "CPU"
- Используйте модели 7B или меньше
- Квантизация Q4_0 или Q2_K
- Установите максимум CPU потоков (равно количеству ядер)

### Рекомендуемые модели для начала:

| Модель | Размер | Применение | Требования VRAM |
|--------|--------|------------|-----------------|
| **Mistral-7B-Instruct** | ~4GB (Q4) | Универсальная, очень хорошая | 6GB+ |
| **Llama-2-7B-Chat** | ~4GB (Q4) | Чат, диалоги | 6GB+ |
| **CodeLlama-7B** | ~4GB (Q4) | Программирование | 6GB+ |
| **TinyLlama-1.1B** | ~700MB (Q4) | Быстрая, для слабого железа | 2GB |
| **Phi-2** | ~1.7GB (Q4) | Компактная но мощная | 3GB |

## 🔍 Поиск моделей

В поиске можно искать:
- По названию: "mistral", "llama", "phi"
- По типу: "chat", "instruct", "code"
- По автору: "TheBloke", "microsoft", "meta"

Сортируйте:
- **По загрузкам** - самые популярные
- **По лайкам** - лучшие по мнению сообщества
- **По дате** - самые свежие

## 🔗 Интеграция с VSCode

1. Запустите сервер в режиме "🔌 API"
2. Установите расширение VSCode (например, [Continue](https://marketplace.visualstudio.com/items?itemName=Continue.continue))
3. В настройках расширения укажите:
   ```json
   {
     "models": [{
       "title": "Local Llama",
       "provider": "openai",
       "model": "local-model",
       "apiBase": "http://localhost:8080/v1"
     }]
   }
   ```
