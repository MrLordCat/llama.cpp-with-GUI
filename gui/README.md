# GUI для llama.cpp

Графический интерфейс для удобной работы с llama.cpp на Windows, Linux и macOS.

## 🚀 Возможности

- **� Запуск llama-server** - полноценный сервер с веб-интерфейсом для чата
- **🔌 Режим API** - интеграция с VSCode, Copilot и другими инструментами
- **🔍 Умный поиск моделей** - поиск по HuggingFace с сортировкой по популярности
- **📥 Загрузка моделей** - скачивание моделей напрямую из HuggingFace
- **🔧 Автоматическая сборка** - сборка llama.cpp с нужным backend (CUDA, Vulkan, Metal, и т.д.)
- **💻 Определение железа** - автоматическое определение GPU и рекомендация оптимальных настроек
- **📦 Установка зависимостей** - помощь в установке необходимых компонентов (Vulkan SDK, CUDA и т.д.)
- **⚙️ Настройка параметров** - все параметры сервера в одном месте
- **💾 Сохранение настроек** - автоматическое сохранение ваших предпочтений

## 📋 Требования

- Python 3.9 или выше
- CMake 3.14+ (для сборки)
- Git

## 🔧 Установка

### 1. Установка Python зависимостей

```bash
# Создайте виртуальное окружение (рекомендуется)
python -m venv venv

# Активируйте его:
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate

# Установите зависимости GUI
pip install -r gui/requirements-gui.txt
```

### 2. Сборка llama.cpp (опционально)

Вы можете собрать llama.cpp прямо из GUI, но если хотите сделать это вручную:

```bash
# Базовая сборка (только CPU)
cmake -B build
cmake --build build --config Release -j

# Сборка с CUDA (для NVIDIA GPU)
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release -j

# Сборка с Vulkan (для AMD GPU)
cmake -B build -DGGML_VULKAN=ON
cmake --build build --config Release -j
```

## 🎮 Запуск

```bash
# Убедитесь, что виртуальное окружение активировано
python gui/llama_gui.py
```

## 📖 Использование

### Вкладка "Запуск сервера"

**Главная особенность**: Запуск llama-server для работы через веб-интерфейс или API!

1. **Выберите модель**:
   - Нажмите "📁 Обзор" и выберите файл `.gguf`
   - Или выберите из списка уже загруженных моделей

2. **Выберите режим работы**:
   - **🌐 Веб-интерфейс** - для чата в браузере (автоматически откроется)
   - **🔌 API режим** - для интеграции с VSCode, GitHub Copilot, и другими инструментами

3. **Настройте параметры сервера**:
   - **Порт**: На каком порту запустить (по умолчанию 8080)
   - **Размер контекста**: Размер окна контекста (рекомендуется 4096)
   - **Потоков**: Количество CPU потоков
   - **Размер батча**: Размер батча для обработки
   - **GPU слоёв**: Сколько слоёв модели загрузить в GPU
   - **CORS**: Включить для веб-доступа
   - **API ключ**: Опционально для защиты доступа

4. **Запустите**: Нажмите "▶️ Запустить сервер"
   - В веб-режиме автоматически откроется браузер
   - В API режиме сервер будет доступен на `http://localhost:8080`

5. **Используйте**:
   - Веб-интерфейс: Чат прямо в браузере
   - API: Подключите к VSCode или другим инструментам

### Вкладка "Загрузка моделей"

**Новинка**: Умный поиск по HuggingFace!

1. **Поиск моделей**:
   - Введите запрос (например: "llama", "mistral", "codellama")
   - Нажмите "🔍 Поиск" или нажмите Enter
   - Или нажмите "⭐ Популярные" для загрузки топ-моделей

2. **Сортировка**:
   - По загрузкам ⬇️ - самые популярные
   - По лайкам ❤️ - самые понравившиеся сообществу
   - По дате обновления 📅 - самые свежие
   - По названию 🔤 - алфавитный порядок

3. **Выбор модели**:
   - Дважды кликните на модель в таблице
   - Выберите конкретный файл из репозитория (разные квантизации)

4. **Скачайте**: Нажмите "📥 Скачать модель"

Модели сохраняются в папку `models/` и автоматически появляются в списках для выбора

### Вкладка "Сборка и настройка"

1. **Выберите backend** в зависимости от вашего GPU:
   - **CPU** - если нет GPU или для тестирования
   - **CUDA** - для NVIDIA GPU (лучшая производительность)
   - **Metal** - для Apple Silicon (M1/M2/M3)
   - **Vulkan** - для AMD GPU или универсальный вариант
   - **SYCL** - для Intel GPU
   - **ROCm** - для AMD GPU (альтернатива Vulkan)

2. **Установите зависимости** (если нужно):
   - Нажмите "📦 Установить зависимости"
   - Следуйте инструкциям для установки Vulkan SDK, CUDA и т.д.

3. **Соберите проект**:
   - Нажмите "⚙️ Конфигурировать" для настройки CMake
   - Нажмите "🔨 Собрать" для компиляции

### Вкладка "Информация о системе"

Показывает:
- Информацию о процессоре
- Объём оперативной памяти
- Обнаруженные GPU
- Рекомендуемый backend для вашего железа

## 🔍 Рекомендуемые модели

### Для начала (требуют мало памяти):

- **Llama-2-7B-Chat-GGUF** - хороший баланс качества и скорости
- **Mistral-7B-Instruct-v0.2-GGUF** - отличное качество на 7B параметрах
- **TinyLlama-1.1B-Chat-GGUF** - очень быстрая, работает даже на слабом железе

### Для более мощных систем:

- **Llama-2-13B-Chat-GGUF** - лучшее качество
- **CodeLlama-13B-Instruct-GGUF** - для программирования
- **OpenHermes-2.5-Mistral-7B-GGUF** - универсальная модель

### Квантизация

При выборе модели обращайте внимание на квантизацию в имени файла:
- `Q2` - самая маленькая, самая быстрая, но хуже качество
- `Q4` - **рекомендуется** - хороший баланс
- `Q5` - лучше качество, немного медленнее
- `Q8` - почти оригинальное качество, требует больше памяти

Пример: `llama-2-7b-chat.Q4_K_M.gguf` - это Llama-2-7B с квантизацией Q4

## ⚠️ Устранение проблем

### "Не найден llama-server.exe"

Сначала соберите проект через вкладку "Сборка и настройка" или вручную через CMake.

### Сервер не запускается / порт занят

Порт 8080 может быть занят другим приложением. Измените порт в настройках сервера (например, на 8081 или 8082).

### Не открывается веб-интерфейс

1. Убедитесь, что сервер запущен (статус должен быть зелёным)
2. Откройте вручную: `http://localhost:8080` (или ваш порт)
3. Проверьте, что CORS включён в настройках

### Как подключить к VSCode?

1. Запустите сервер в API режиме
2. Установите расширение для работы с LLM (например, Continue)
3. Настройте URL: `http://localhost:8080` (или ваш порт)
4. Если установлен API ключ, добавьте его в настройки расширения

### "Не хватает памяти GPU"

Уменьшите количество GPU слоёв в параметрах. Начните с малых значений (5-10) и увеличивайте.

### "Модель загружается очень долго"

Это нормально для больших моделей (13B+). Дождитесь окончания загрузки или выберите модель поменьше.

### Проблемы с Vulkan на AMD GPU

Убедитесь, что установлен Vulkan SDK:
- Windows: [https://vulkan.lunarg.com/sdk/home#windows](https://vulkan.lunarg.com/sdk/home#windows)
- Проверьте переменную окружения `VULKAN_SDK`

### Медленная генерация на CPU

Это нормально. CPU значительно медленнее GPU. Используйте:
- Меньшие модели (7B вместо 13B)
- Больше CPU потоков (равно количеству ядер)
- Квантизацию Q4 или Q2
